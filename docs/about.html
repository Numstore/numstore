What is Numstore?

Numstore is a database designed specifically for contiguous byte streams. It combines the flexibility of a file system with the performance and reliability guarantees of a database.

At its core, Numstore uses an R+Tree data structure - a rope equivalent to a B+Tree that maintains byte count information at each node. This architecture enables logarithmic time complexity for insertions, deletions, and reads at any position in the byte stream.

Context and Motivation

Traditional file systems provide linear access to data, making insertions and deletions in the middle of a file expensive O(N) operations. Databases offer better random access but often aren't optimized for contiguous byte operations.

Numstore bridges this gap by providing:

  • O(log N) insertions and deletions anywhere in the stream
  • Database-grade reliability with ACID transactions
  • High throughput (640+ MB/s sustained writes)
  • A flexible type system for structured data

Historical Precedent

Machine Learning and Scientific Computing

Scientific computing and machine learning workflows often require efficient random access to large arrays of structured data. Formats like HDF5 and NetCDF address some of these needs but don't provide efficient mid-stream insertions or database-style transactions.

US Intelligence Community

The intelligence community has long worked with massive datasets requiring efficient random access, structured storage, and strong reliability guarantees. Numstore's design draws inspiration from these requirements.

Other Scientific and Engineering Formats

Numstore complements existing formats like:

  • HDF5 - Hierarchical data format for scientific data
  • NetCDF - Network Common Data Form for array-oriented data
  • FITS - Flexible Image Transport System for astronomy
  • Parquet - Columnar storage for analytics

Unlike these formats, Numstore prioritizes efficient mid-stream mutations and provides transaction support with crash recovery.

Numstore's Positioning

Numstore is designed for applications that need:

  • Efficient random insertions/deletions in large byte streams
  • Strong reliability guarantees (ACID, crash recovery)
  • Structured data with a type system
  • High throughput for both reads and writes
  • Simple, file-like interface with database features

Numstore is NOT designed for:

  • Relational data with complex queries (use PostgreSQL, MySQL)
  • Key-value lookups (use Redis, RocksDB)
  • Document storage (use MongoDB, CouchDB)
  • Distributed/replicated storage (use Cassandra, Riak)

Use Cases

Ideal applications for Numstore include:

  • Time-series databases - Efficient append and random read operations
  • Scientific datasets - Large arrays with structured types
  • Machine learning pipelines - Training data with complex access patterns
  • Engineering simulations - Large contiguous data structures
  • Log aggregation - High-throughput writes with reliable persistence
  • Sensor data - Continuous streams with structured records
