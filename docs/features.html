Core Features

High Performance Operations

All core operations execute in logarithmic time complexity:

  • O(log N) Insertions - Insert data anywhere in the byte stream
  • O(log N) Deletions - Remove data from any position
  • O(log N) Reads - Access data at any offset
  • 640+ MB/s Write Speed - Sustained high-speed writes

Persistent Storage

  • Automatic persistence with write-ahead logging (WAL)
  • ARIES-based undo/redo logging
  • Crash recovery with automatic rollback
  • Durable ACID transactions

Flexible Data Access

  • Strided read/write operations (e.g., read every 3rd element)
  • Offset-based access patterns
  • Range operations (start:stop:stride)
  • Direct TCP socket streaming for large data transfers

Comprehensive Type System

  • Primitives - Signed/unsigned integers, floats, complex numbers
  • Structs - Composite data structures
  • Unions - Variant types
  • Arrays - Fixed-size arrays of any type
  • Enums - Enumerated types

Multi-Language Support

  • Native C library (libnscore, libnspager)
  • Python wrapper with Pythonic API
  • Java bindings for JVM integration
  • Command-line interface for scripting

R+Tree Architecture

Numstore is built on an R+Tree data structure - a rope equivalent to a B+Tree that tracks byte counts per node. This architecture enables:

  • Efficient random access to any position in the byte stream
  • Fast insertions and deletions without full data reorganization
  • Optimal cache performance with node-level operations
  • Scalable storage for large datasets

Transaction Support

Full ACID transaction guarantees:

  • Atomicity - All operations in a transaction succeed or fail together
  • Consistency - Database remains in valid state after transactions
  • Isolation - Concurrent transactions don't interfere
  • Durability - Committed transactions survive system failures

What Numstore Can Do

  • Store and manipulate large contiguous byte streams efficiently
  • Handle time-series data with fast insertions and queries
  • Manage scientific datasets with complex access patterns
  • Provide file-system-like operations with database guarantees
  • Support machine learning workflows with typed array storage

What Numstore Cannot Do

Numstore is specialized for contiguous byte operations and has limitations:

  • Not a general-purpose relational database (no SQL)
  • Not optimized for highly sparse data structures
  • Not designed for document or key-value storage patterns
  • Not a distributed database (single-node only)
