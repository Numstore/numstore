Prerequisites

  • C Compiler (GCC or Clang)
  • CMake 3.10 or higher
  • Git (for cloning the repository)

Building from Source

Linux

# Install dependencies (Ubuntu/Debian)
sudo apt-get update
sudo apt-get install build-essential cmake git

# Clone the repository
git clone https://gitlab.com/lincketheo/numstore.git
cd numstore

# Build the project
make

# Run tests
./build/debug/apps/test

macOS

# Install dependencies using Homebrew
brew install cmake

# Clone the repository
git clone https://gitlab.com/lincketheo/numstore.git
cd numstore

# Build the project
make

# Run tests
./build/debug/apps/test

Windows

# Install CMake and a C compiler (MinGW or MSVC)
# Clone the repository
git clone https://gitlab.com/lincketheo/numstore.git
cd numstore

# Build using CMake
cmake -B build
cmake --build build

# Run tests
.\build\Debug\apps\test.exe

Main Concepts

Before diving into using Numstore, it's important to understand these core concepts:

Variables

In Numstore, a variable is a named container for typed data. Variables have a type definition and can store arrays of that type.

Types

Numstore supports a comprehensive type system including primitives, structs, unions, arrays, and enums. See the Type System documentation for details.

Operations

Core operations on variables include:

  • Create - Create a new variable with a type
  • Insert - Insert elements at a specific position
  • Append - Add elements to the end
  • Read - Read elements from a range
  • Write - Update existing elements
  • Remove - Delete elements from a range
  • Take - Read and remove elements atomically
  • Delete - Delete a variable entirely

Quick Example

Here's a simple example to get you started:

# Create a variable with a struct type
./nstore "create data struct { x i32, y f64 }"

# Insert data (via stdin)
echo -e "\\x01\\x00\\x00\\x00\\x00\\x00\\x00\\x00\\x00\\x00\\xf0\\x3f" | \
  ./nstore "insert data 0"

# Read the data back
./nstore "read data 0:1:1" | hexdump -C

Running Examples

Numstore includes 15+ comprehensive examples. To run them:

# Build examples
make

# Run a basic persistence example
./build/debug/apps/examples/nsfslite/example1_basic_persistence

# List all available examples
ls ./build/debug/apps/examples/nsfslite/
